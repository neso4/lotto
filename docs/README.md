## 3주차 목표
1. 에러 메시지를 출력 후 해당 부분부터 재입력
2. 클래스(객체)를 분리하는 연습
3. 도메인 로직에 대한 단위 테스트를 작성하는 연습
4. Java Enum 사용
5. 도메인 로직에 단위 테스트를 구현

## 설계하기
### 요청
- 현금 요청
- 현금만큼 로또사기 요청
- 6개의 로또번호 요청
- 당첨번호와 보너스 번호 요청
- 당첨번호와 보너스 번호 비교 요청
- 수익률 비교요청

### 역할
- 사용자
- 컴퓨터(랜덤번호를 뽑기)
- Lotto
- 당첨번호
- 당첨 결과


## 기능 요구사항 정리

### Model
1. Lotto
    
    <상태>
    - [x] 로또 번호 리스트
    
   <행동>
    - [x] 6번호의 숫자 검사
    - [x] 6번호의 숫자 중복검사
    - [x] 오름차순 정렬
    - [x] 입력 받은 당첨번호를 리스트 형태로 변환
    - [x] 구매한 로또 번호와 당첨 번호 비교

2. LottoMachine

    <상태>
    - [x] 랜덤 숫자 리스트
   
   <행위>
    - [x] 랜덤 숫자 6개 뽑기


3. User
   
    <상태>
   - [x] 구매 로또 번호 리스트
   - [x] 현금

    <행동>
   - [x] 구매한 로또 리스트에 저장하기
   
4. LottoWinningNumbers
    
    <상태>
    - [x] 당첨로또 번호
    - [x] 당첨 보너스 번호
    - [x] 로또 결과

    <행동>
    - [x] 당첨로또번호와 사용자의 로또 비교
    - [x] 로또 결과의 등수별 개수와 총 수익 요청  


5. LottoResult(당첨된 결과 종합)
    
    <상태>
   - [x] 당첨된 등수의 개수
   - [x] 당첨된 총수익
   - [x] 수익률

    <행동>
   - [x] 당첨등수 적합 검사 
   - [x] 당첨된 등수의 개수 +1
   - [x] 당첨된 등수에 맞게 수익 더하기
   - [x] 총 수익률 구하기
   

### View
1. InputView
  - [x] 예외발생시 오류 출력 및 재입력
    - [x] 구입금액 입력 메시지
    - [x] 당첨번호 입력 메시지
    - [x] 보너스번호 입력 메시지

2. OutputView
   - [x] 로또 구매 수 메시지
   - [x] 구매한 로또 번호 메시지
   - [x] 당첨통계 메시지
   - [x] 수익률 메시지

### Controller
1. LottoController
   - [x] 로또 게임시작
   - [x] 게임 종료

### Validation
1. InputValidation
   - [x] 구입금액의 데이터 검사
     - [x] 빈문자열 검사
     - [x] 숫자 판별 검사
     - [x] 21억 이하 검사
     - [x] 0이하 검사
     - [x] 1000단위 검사
   - [x] 당첨번호 검사
     - [x] 빈문자열 검사
     - [x] 1~45사이의 번호 검사
     - [x] 6개검사
     - [x] 중복검사
   - [x] 보너스번호 검사
     - [x] 빈문자열 검사
     - [x] 숫자 판별 검사
     - [x] 1~45사이의 번호 검사
     - [x] 중복 검사

### Enum(상수)
1. LottoPrize
    - 각각의 당첨을 위한 개수와 금액 상수 처리
    - 당첨등수에 따른 상금 제공
