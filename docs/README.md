# 💵 기능 구현 목록

# 1. 로또 구입 금액 입력 받도록 요구하는 출력(Enum관리)

- “구입금액을 입력해 주세요”

# 2. 사용자의 로또 구입액 입력

- 1000원 단위로 떨어지는지 체크
    - 1000원 단위로 떨어지지 않으면 예외 발생
- 숫자가 아닐 경우 예외 발생
- 1000원 보다 작을 경우 예외 발생

# 3. 사용자의 로또 구매 횟수 출력(Enum관리)

- “`구매횟수` 개를 구매했습니다.” 출력

# 4. `구매횟수` 번 만큼 사용자의 로또 번호 출력

- 각 로또 번호 생성
    - `pickUniqueNumbersInRange()` 사용
- `List<Lotto>lotto` 형식으로 구현

# 5. 당첨 번호 입력 받도록 요구하는 출력(Enum관리)

- “당첨 번호를 입력해 주세요.”

# 6. 당첨 번호 입력

- 당첨 번호 클래스 생성
- 당첨 번호 입력 유효성 검사
    - 시작이 숫자인지 검사 + 숫자 인지 아닌지 검사
    - 중간에 공백이 있는지 검사
    - 마지막에 “,”가 있는지 검사
    - 6개의 숫자인지 검사
    - 1 ~ 45인지 검사
- 숫자 파싱해 List 저장

# 7.  보너스 번호 입력 요구하는 출력(Enum관리)

- “보너스 번호를 입력해 주세요.”

# 8. 보너스 번호 입력

- 입력된 번호 유효성 검사
    - 당첨번호 + 보너스 번호 중복 체크
    - 숫자인지 체크
    - 1 ~ 45 인지 체크

# 9. “당첨 통계” 출력(Enum관리)

- “당첨통계” 출력
- “- - -” 출력

# 10. 로또 일치 여부 출력

- 사용자의 로또 번호와 당첨 번호와 일치하는지 검사
    - 일반적인 당첨번호 일치
    - 5개의 일치 번호 중 보너스 번호와 일치하는지 검사

# 11. 수익률 계산 및 출력

- 처음에 사용자의 구입 금액과 수익금 비교
