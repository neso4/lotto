## 🚀 기능 구현 사항

### 두 개의 클래스로 구현

```
- 로또 번호를 저장하는 클래스, 로또 당첨을 확인하는 클래스
- 클래스 명은 각각 Lotto, LottoGame
- 전체 코드를 실행하는 메소드는 LottoGame.run()
```

---

## 1. Lotto

### (1) 필드

```
로또 번호를 저장할 List
```

### (2) 메소드

#### 0) Lotto 클래스의 생성자

```
- 로또 번호의 개수가 6인지 확인
- 중복된 로또 번호 확인
- 로또 번호 저장
```

#### 1) validate

```
- 생성한 로또 번호의 개수가 6인지 확인하는 메소드
```

#### 2) checkDuplicatedNumber

```
- 중복된 로또 번호가 있는지 확인하는 메소드
```

#### 3) printNumbers

```
- 로또 번호를 출력하는 메소드
```

#### 4) printNumbers

```
- 로또 번호를 반환하는 메소드
```

## 2. LottoGame

### 1. 필드

#### 1) winning

```
- 로또 당첨 번호를 저장하는 리스트
```

#### 2) lottos

```
- 구입한 여러 로또 번호를 저장하는 리스트
```

#### 3) price

```
- 구입할 로또 금액을 저장하는 변수
```

#### 4) bonus

```
- 로또 보너스 번호를 저장하는 변수
```

#### 5) lottoCount

```
- 구입한 로또 장수를 저장하는 변수
```

#### 6) winCount

```
- 일치하는 번호 수에 따라 횟수를 저장하는 맵
```

#### 7) sumPrice

```
- 당첨금의 합을 저장하는 변수
```

### 1. 메소드

#### 0) lottoGame 클래스의 생성자

```
- lottoGame의 필드를 초기화하는 단계
```

#### 1) inputPrice

```
- 로또 구입 금액을 입력하고 구입한 로또를 출력하는 단계
- 잘못된 입력이 발생하면 예외를 처리하고 에러메시지 출력
```

#### 2) enterPrice

```
- 로또 구입 금액을 입력하는 단계
- 공백을 제거하고, 정수 이외에 문자가 입력되었는지 아니면 음수가 입력되었는지 확인한다.
- 만약 정수 이외의 문자가 입력되었거나, 음수가 입력되었으면 예외 발생
```

#### 3) checkNegativePrice

```
- 음수가 입력되면 예외를 발생시키는 메소드
```

#### 4) setLottoCount

```
- 입력한 구입 금액을 토대로 로또 발행 수를 계산
- 로또 번호의 개수가 0 이하인지 확인
```

#### 5) checkLottoCount

```
- 로또 발행 수가 0 이하면 예외를 발생시키는 메소드 
```

#### 6) setLotto

```
- 로또 번호를 무작위로 생성하고, 오름차순으로 정렬한다.
- 생성한 로또 번호를 매개변수로 하여 Lotto 클래스의 인스턴스 생성
```

#### 7) makeLottos

```
- 로또 발행 수 만큼 Lotto 인스턴스를 생성 후 리스트에 저장한다.
```

#### 8) printLottos

```
- 로또 발행 수를 출력한다.
- 발행한 로또 번호를 출력한다.
```

#### 9) inputWinningNumber

```
- 로또 당첨 번호를 입력한다.
- 잘못된 당첨번호를 입력하면 예외를 처리하고, 에러 메시지를 출력한다.
```

#### 10) enterWinningNumber

```
- 로또 당첨 번호를 입력한다.
- 입력한 번호에서 공백을 제거한다.
- 정수와 쉽표 외의 문자가 입력되었는지 확인한다.
- 번호를 분리하고, 리스트에 저장한다.
- 당첨 번호의 개수가 6개인지 확인한다.
- 중복된 당첨 번호가 있는지 확인한다.
```

#### 11) deleteSpace

```
- 입력한 문자열에서 공백을 제거하는 메소드
```

#### 12) checkRangeNumber

```
- 로또 번호의 범위가 1 ~ 45가 아니면 예외 발생
```

#### 13) checkWinningLength

```
- 당첨 번호의 개수가 6개가 아니면 예외 발생
```

#### 14) checkSpecialCharacter

```
- 입력받은 문자열에서 정수와 쉼표 이외의 문자가 있으면 예외 발생
```

#### 15) checkDuplicateWinNumber

```
- 당첨 번호에서 중복된 번호가 존재하면 예외 발생
```

#### 16) inputBonusNumber

```
- 보너스 번호를 입력한다.
- 잘못된 보너스 번호가 입력되면 예외를 처리하고 에러 메시지를 출력한다.
```

#### 17) enterBonusNumber

```
- 보너스 번호를 입력한다.
- 공백을 제거하고, 정수로 반환한다. 이때 정수가 아니면 예외가 발생한다.
- 보너스 번호가 범위 안에 있는지 확인한다.
- 당첨번호와 중복되는지 확인한다.
```

#### 18) checkDuplicateBonus

```
- 보너스 번호가 당첨 번호와 중복되면 예외 발생
```

#### 19) initWinCount

```
- 당첨 횟수를 저장하기 위한 맵을 초기화 한다.
```

#### 20) calcWin

```
- 로또 번호와 당첨 번호를 확인하여 일치하는 번호 개수를 구한다.
- 일치하는 번호 개수가 3 ~ 6개인지, 그리고 번호 개수가 5개 일치할 때 보너스 번호까지 일치하는지 확인하고, 맵에 당첨 횟수를 저장한다.
```

#### 21) printResult

```
- 각 로또 별로 당첨 여부를 확인한다.
- 당첨 통계를 출력한다.
- 총 수익률을 출력한다.
```

#### 22) calcTotalWin

```
- 각 로또 별로 당첨 여부를 확인한다.
```

#### 23) printWin

```
- 당첨 통계를 출력한다.
- 일치 개수가 3 ~ 6인 로또의 개수를 출력한다.
```

#### 24) printTotal

```
- 당첨금에서 로또 구입 금액을 나눠 총 수익률을 계산 후 출력한다.
```

#### 25) run

```
- 위에서 구현한 메소드를 조합하여 로또 게임을 구성하고 실행한다.
```

### 1. LottoTest

#### 2) createLottoByDuplicatedNumber


#### 4) checkGetNumber

```
- 생성한 로또 번호를 그대로 반환하는지 확인한다.
```

### 2. ApplicationTest

#### 1) 기능_테스트

```
- 로또 게임이 제 기능을 다하는지 확인한다.
- 첫 번째 테스트는 8000원의 구입 금액과 당첨번호 1,2,3,4,5,6과 보너스 번호 7을 입력하고 원하는 결과가 나오는지 확인한다.
- 두 번째 테스트는 로또 번호가 생성되었을 때 정렬되어서 출력하는지 확인한다.
- 세 번째 테스트부터 일치 개수가 4 ~ 6개 까지 제대로 출력되는지 확인한다.
```

#### 2) 예외_테스트

```
- 구입 금액, 로또 당첨 번호, 보너스 번호 입력에 대해서 발생할 수 있는 예외에 대하여 테스트를 진행한다.
- 구입 금액 테스트에서는 숫자 이외의 문자 입력, 음수 입력, 1000원 미만의 입력에 대해서 예외가 발생하는지 확인한다.
- 로또 당첨 번호 테스트에 대해서는 번호 개수가 6이 아니거나, 구분자가 쉼표가 아니거나, 중복된 번호가 있거나, 숫자가 아닌 문자가 포함되어있거나, 범위가 벗어난 숫자가 입력되었을 때 예외가 발생하는지 확인한다.
- 보너스 번호 테스트에 대해서는 숫자가 아닌 문자 입력, 범위를 벗어난 숫자 입력, 당첨 번호와 중복된 번호가 입력되었을 때 예외가 발생하는지 확인한다.
```
