# 3주 차 미션 - 로또 🎰
## <기능구현 목록>

### 문제
- 로또 게임 기능을 구현해야 한다.

### 규칙

- 당첨은 1등부터 5등까지 있다. 당첨 기준과 금액은 아래와 같다.
  - 1등: 6개 번호 일치 / 2,000,000,000원
  - 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
  - 3등: 5개 번호 일치 / 1,500,000원
  - 4등: 4개 번호 일치 / 50,000원
  - 5등: 3개 번호 일치 / 5,000원
- 로또 1장의 가격은 1000원이다.
- 로또 구입 금액을 입력하면 구입 금액에 해당하는 만큼 자동으로 로또 숫자를 발행한다.
- 당첨 번호와 보너스 번호를 사용자에게 입력받는다.
- 사용자가 구매한 로또 번호와 당첨 번호를 비교하여 당첨 내용 및 수익률을 출력하고 로또 게임을 종료한다.
- 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException를 발생시키고, "[ERROR]"로 시작하는 에러 메시지를 출력 후 그 부분부터 입력을 다시 받는다.
- Exception이 아닌 IllegalArgumentException, IllegalStateException 등과 같은 명확한 유형을 처리한다.


### 1, 로또 번호 랜덤 생성
- 로또 번호의 숫자 범위는 1~45까지이다.
- 1개의 로또를 발행할 때 중복되지 않는 6개의 숫자를 뽑는다.

### 2, 사용자 입력
- 로또 구입 금액을 입력받는다. 구입 금액은 1,000원 단위로 입력받는다.
- 당첨 번호를 입력받는다. 번호는 쉼표(,)를 기준으로 구분한다.

### 3,요구 기능
- 로또 구입 금액을 입력받는 기능
- 당첨 번호 6자리와 보너스 번호 1개를 증복없이 입력받는 기능
- 로또 구입 금액만큼 생성하는 기능
- 로또 하나당 6개의 번호를 로또 번호를 램덤 생성하는 기능
- 생성된 로또 개수와 각각의 번호 6개를 정렬하고 출력하는 기능
- 생성된 로또마다 당첨 등수를 확인하고 개수를 저장하는 기능
- 당첨 내용을 출력하는 기능
- 구입 금액과 당첨 금액을 비교하여 수익률을 계산하는 기능


### 5, 예외 사항
- 사용자 입력 로또 구입 금액이 1000원으로 나누어떨어지지 않는 경우
- 사용자 입력 입력한 금액이 숫자가 아닐 경우
- 사용자 입력 1~45의 범위를 벗어나거나 숫자가 아닌 경우
- 사용자 입력 로또 당첨 번호의 개수가 6 아닐 때
- 사용자 입력 보너스 번호 개수가 1이 아닌 경우
- 사용자 입력 로또 당첨 번호와 보너스번호에 중복인 수가 있을 경우
